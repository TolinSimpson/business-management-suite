Here is a blank project. Plan a schedule app that runs on android or web. We want to keep the code minimal and bloat free. The phone must notify the user each hour when it is time to move onto the next task in the day. The user may keep their phone on power savings mode all day so push notifications need to bypass that or another system used. I.e. blocking out the screen during certain time blocks and playing audio on schedule change. The user will be managing LLM coding agents during the day so most tasks will involve prompting and confirmation. Ideally the user wants to switch to happy.engineering to do this from a phone without a laptop. Keep the UX clean and simple, ensure it is easy to plan the tasks for the week/day during the task planning time-block while keeping the current general shedule goals intact. i.e. It is 10:00am on a monday display Deep work - Planning phase - specific task if set. Make sure that the schedule is shareable to others via a link with share options i.e. detail levels to include. After each hour time block I'd like to have checkboxes to check off parts of the process. i.e. Planning: [X] Prayed [X] Defined Goals [X] drafted plan [X] risk assessment

For context the end user is a software developer and Christian with a tight routine. We want to keep weekends restful and work days productive with clear objectives for the day. The user would eventually like to transition to a fractional multi-client work week where time blocks or days are auctioned off but currently works a normal 9-5.
